THE CALM CTO

New junior onboarding timeline

Week 1-2
Understand first
  • • System architecture deep-dive
  • • Read existing PRs
  • • Explain what they do
  • • Watch AI tools, don't use yet
Week 3-4
Use AI with accountability
  • • Small, well-defined changes
  • • Ship the feature AND
  • • Explain every AI-generated line
  • • Full code review required
Month 2-3
Build judgment
  • • Progressively harder features
  • • AI helpful but not sufficient
  • • When to trust AI output
  • • When to ignore it entirely
This takes more senior time. That's unavoidable.
